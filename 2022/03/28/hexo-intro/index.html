GitHub-Page+Hexo创建个人网站

GitHub Page + Hexo搭建静态网站

​ 用过worldPress和halo，需要自己搭服务器，花钱而且麻烦，如果是写点简单的笔记的话，可以考虑github page + hexo来搭建一个静态博客。

1、环境准备

  • git
  • nodejs
  • hexo
  • hexo theme包

2、搭建步骤

2.1、安装上述环境

git，nodejs安装如常，hexo使用npm安装 ， 注意npm版本。

2.2、创建git仓库

进入github（如没有账户，需先注册），创建一个仓库repository， 注意仓库名youUsername.github.io，这里的youUsername 就是你的github用户名；

2.3、获取你的仓库的完整地址

如https://github.com/likepepper/likepepper.github.io.git，后面的配置会用到；

2.4、初始化一个本地hexo项目

在d盘创建一个hexo文件夹，控制台输入hexo init projectName，进行项目的初始化；

2.5、站点配置

上一步初始化后，文件夹中会多出一些文件，其中_config.yml 为网站的配置，最后一行，加上部署的方式，类型为git，地址为2.3中的网站仓库地址，

还有其他一些配置，根据自己的需要进行页面的定制；

2.6、主题配置

主题获取地址：https://hexo.io/themes/

将你喜欢的地址引入，建议下载到本地引入，因为需要进行自定义的修改，也可以在hexo项目目录下使用npm引入主题，我这里是直接下载到hexo项目的/theme/文件夹下使用的，在/theme/文件夹下，同样的有一个_config.yml ，这个是主题配置文件，和上面的网站配置文件区分开；

我这里使用的主题是Stellar，不同的主题配置不同，按照主题提供者的配置说明进行配置；

2.7、同步页面

配置完成（需要重点检查，站点配置的地址，容易出错），在本地运行检查无误后，控制台使用hexo clean 先清理一下，然后hexo g 将配置好的网站生成静态的文件，最后使用hexo d 进行部署，部署会将静态文件上传到git仓库

2.8、github设置

进入github，确认网站的静态文件都上传到仓库了，点击仓库的Settings ，找到Pages 设置，在source 中设置仓库的网站所在分支，和文件夹，我是使用的默认的main分支，和根目录/(root)

，点击save 进行保存，如果提示：

Your site is published at https://yourUsername.github.io/

表示你的GitHub Page已经生成了，可以直接点击链接进行访问。
